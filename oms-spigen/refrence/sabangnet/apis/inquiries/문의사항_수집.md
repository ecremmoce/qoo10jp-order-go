# 문의사항 수집
**분류**: inquiries

## RESTful API 변환

**Method**: `POST`

**Path**: `/api/v1/inquiries/collect`

**Summary**: 문의사항 수집

## 인증 (Authentication)

### 사방넷 XML 헤더

| 필드명 | 설명 | 필수 |
|--------|------|------|
| SEND_COMPAYNY_ID | 사방넷 로그인아이디 | Y |

### RESTful HTTP 헤더

```http
Authorization: Bearer {API_KEY}
X-Company-ID: {COMPANY_ID}
Content-Type: application/json
```

## 헤더 필드 (Header Fields)

| 필드명 | 설명 | 필수 | 비고 |
|--------|------|------|------|
| SEND_COMPAYNY_ID | 사방넷 로그인아이디 | Y | 사방넷 어드민 로그인 아이디입력 |

## 데이터 필드 (Data Fields)

| XML 필드명 | JSON 필드명 | 설명 | 필수 | 비고 |
|-----------|-------------|------|------|------|
| CS_ST_DATE | csStDate | 검색 시작일자 | Y | YYYYMMDD : 수집일자 기준 검색 시작일 |
| CS_ED_DATE | csEdDate | 검색 마지막일자 | Y | YYYYMMDD : 수집일자 기준 검색 종료일 |
| CS_STATUS | csStatus | 검색 처리구분 | N | 001:신규접수, 002:답변저장, 003:답변전송, 004:강제완료, NULL:전체 |
| 비고 | 비고 | - | N | - |
| 문의 등록(수집시) 사방넷에서 부여되는 유니크번호 | 문의 등록(수집시) 사방넷에서 부여되는 유니크번호 | - | N | - |
| 수집된 문의사항의 쇼핑몰명 | 수집된 문의사항의 쇼핑몰명 | - | N | - |
| 수집된 문의사항의 쇼핑몰 로그인아이디 | 수집된 문의사항의 쇼핑몰 로그인아이디 | - | N | - |
| 001:신규접수, 002:답변저장, 003:답변전송, 004:강제완료 | 001:신규접수, 002:답변저장, 003:답변전송, 004:강제완료 | - | N | - |
| 수집일자 | 수집일자 | - | N | - |
| 쇼핑몰 주문번호 | 쇼핑몰 주문번호 | - | N | - |
| 쇼핑몰 상품코드 | 쇼핑몰 상품코드 | - | N | - |
| 질문자 | 질문자 | - | N | - |
| 질문일자 | 질문일자 | - | N | - |
| 문의구분 : 상품문의, Q&A,긴급메시지,... | 문의구분 : 상품문의, q&a,긴급메시지,... | - | N | - |
| 품번코드가 존재 할 경우 해당 품번의 자체상품코드 | 품번코드가 존재 할 경우 해당 품번의 자체상품코드 | - | N | - |
| 수집된 CS의 쇼핑몰 코드 | 수집된 cs의 쇼핑몰 코드 | - | N | - |

```xml

  <HEADER>
    <SEND_COMPAYNY_ID>사방넷 로그인아이디</SEND_COMPAYNY_ID>
  </HEADER>
  <DATA>
    <ITEM>
      <CS_ST_DATE>값</CS_ST_DATE>
      <CS_ED_DATE>값</CS_ED_DATE>
      <CS_STATUS>값</CS_STATUS>
      <비고>값</비고>
      <문의 등록(수집시) 사방넷에서 부여되는 유니크번호>값</문의 등록(수집시) 사방넷에서 부여되는 유니크번호>
    </ITEM>
  </DATA>

```
